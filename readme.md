# Инструмент визуализации графа зависимостей (Вариант №1)

## Цель работы

Разработать Python-инструмент, который самостоятельно строит и визуализирует граф зависимостей Python-пакетов без использования pip, сторонних библиотек анализа зависимостей или готовых менеджеров пакетов.

## Что реализовано

1. CLI-приложение с параметрами командной строки:

  – имя пакета
  – версия
  – URL репозитория или путь к тестовому репозиторию
  – режим тестирования
  – максимальная глубина анализа

2. Обработка ошибок:

  – отсутствующие параметры
  – неправильный формат версии
  – недоступный URL или файл
  – некорректные данные тестового репозитория

3. Получение прямых зависимостей Python-пакетов
  – ручный парсинг метаданных пакета (METADATA/PKG-INFO)

4. Построение полного графа зависимостей:

  – алгоритм DFS с рекурсией
  – учёт максимальной глубины
  – защита от циклических зависимостей
  – режим тестового репозитория (буквенные пакеты)

5. Порядок загрузки зависимостей
  – топологическая сортировка

6. Визуализация графа
  – генерация диаграммы в формате PlantUML

## **Инструкция по использованию**

### **Запуск программы**

```
python <название файла> --package <имя пакета> --test-repo --repo-file <название файла с репозиторием> --ascii-tree --print-order --plantuml
```
